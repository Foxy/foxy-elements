# Card Embed

Developer diagnostics suite for foxy-payment-card-field. These stories prioritize maintainers and AI agents validating runtime behavior and integration contracts.

## Story Selection Guide

- Validate reflected API and attributes: use ApiPlayground
- Verify mode reflection and runtime switching: use ModeTransitions
- Reproduce validity transitions: use ValidationStates
- Verify theme attribute passthrough: use ThemeAttributeControls
- Verify deterministic tokenize success contract: use TokenizeSuccess
- Replay tokenize failure path deterministically: use TokenizeErrorMatrix
- Exercise form-associated integration: use FormIntegrationInteraction
- Verify disabled host state styling: use Disabled

### States

The element exposes ElementInternals custom states on the host:

- :state(focused)
- :state(user-invalid)
- :state(user-valid)
- :state(disabled)

Native :valid and :invalid also apply because the element is form-associated.

## Local Dev Notes

- The hosted iframe is loaded from VITE_EMBED_ORIGIN.
- If that origin is unavailable, stories still validate host API, controls, events, and deterministic play interactions, but browser console network errors for iframe requests are expected.
