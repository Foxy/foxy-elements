# ACH Field

Developer diagnostics suite for `foxy-ach-field`. These stories prioritize maintainers and AI agents working on runtime behavior, not consumer onboarding.

## Story Selection Guide

- Validate reflected API and attributes: use `ApiPlayground`
- Verify focus/change/blur semantics: use `EventSemantics`
- Reproduce validity transitions: use `ValidityStates`
- Check same-group vs isolated session behavior: use `SessionGroupTopology`
- Verify tokenize success payload contract: use `TokenizeSuccess`
- Replay tokenize error codes deterministically: use `TokenizeErrorMatrix`
- Validate account type restriction wiring: use `AccountTypeRestrictions`
- Verify theme attribute passthrough: use `ThemeAttributeControls`
- Exercise minimal form-associated integration flow: use `FormIntegrationInteraction`

### States

The element exposes `ElementInternals` custom states on the host:

- `:state(focused)`
- `:state(user-invalid)`
- `:state(user-valid)`
- `:state(disabled)`

Native `:valid` and `:invalid` also apply because the element is form-associated.

## Local Dev Notes

- The hosted iframe payload is served by the ACH embed origin configured in `VITE_EMBED_ORIGIN`.
- If that origin is unavailable, stories still validate host-level API behavior, controls, events, and play interactions, but you may see network errors in the browser console for iframe requests.
