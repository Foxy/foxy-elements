# ACH Field

Renders a secure hosted ACH input inside an iframe. You can use this element to securely collect ACH data on your website or web app. Every element renders only one field, without label or helper text. You can render as many fields as you need as style them to look like the regular input fields in your project.

## Setup

To use the ACH element, include the script and render the fields you need in a standard web form. The element is form-associated and contributes native validity, so you can use it with native form submission or any custom flow that fits your needs.

When the form is submitted, call `tokenize` on any field to tokenize all fields in the same session. The resulting token can be used to process ACH payments on your server.

```html
<form id="ach-form">
  <label>Routing number: <foxy-ach-field field="routing_number"></foxy-ach-field></label>
  <label>Account number: <foxy-ach-field field="account_number"></foxy-ach-field></label>
  <label>Account type: <foxy-ach-field field="account_type"></foxy-ach-field></label>
  <label>Name on account: <foxy-ach-field field="account_holder_name"></foxy-ach-field></label>

  <button type="submit">Pay</button>
</form>

<script type="module">
  import "https://cdn-js.foxy.io/elements@2/foxy-ach-field.js?store=my-store";

  const form = document.getElementById("ach-form");
  const field = form.querySelector("foxy-ach-field");

  async function submit() {
    try {
      const result = await field.tokenize();
      // Send result.token to your server to process the payment
    } catch (error) {
      console.error("ACH tokenization failed", error);
    }
  }

  form.addEventListener("submit", (event) => {
    event.preventDefault();
    if (form.reportValidity()) submit();
  });
</script>
```

## Methods

### `clear(): void`

Clears ACH values in the current session.

### `tokenize(requestId?: string): Promise<{ token: string; last4: string; bankName?: string; requestId?: string }>`

Tokenizes session fields.

### `checkValidity(): boolean`

Uses form-associated validity state.

### `reportValidity(): boolean`

Uses native form validation UI/flow.
